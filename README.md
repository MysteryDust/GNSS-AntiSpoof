# GNSS-AntiSpoof — Real-Time Standalone Positioning with Subset-Spoofing Detection & Mitigation

> Intro to Navigation (2026b) — Course Project, **Option 1**
> *"Consider Ex0, design a real-time navigation (Android) app that computes the standalone
> positioning via raw measurements — allowing detection and overcoming spoofing of a subset
> of the navigation satellites."*

This project extends **Ex0** (offline RINEX → KML/CSV positioning) into a **real-time engine**
that:

1. computes a **standalone** Position/Velocity/Time (PVT) solution directly from **raw GNSS
   measurements** — bypassing the Android OS / Fused Location Provider, and therefore immune to
   mock-location and Wi-Fi/network spoofing **by construction**;
2. **detects** spoofing of a *subset* of the navigation satellites, using a dual layer of
   **geometric** (RANSAC + RAIM) and **physical** (AGC / C-N₀ Receiver Power Monitoring) checks;
3. **overcomes** the spoofing — it names the spoofed satellites, excludes them, and recomputes a
   **trusted** position from the authentic subset; and when the spoof is a *majority*, it falls
   back to a position-prior gate + dead reckoning so the reported position stays protected.

The core is **pure-Python (standard library only)** — nothing to `pip install`. A companion
**Kotlin Android module** (`android/`) mirrors the same algorithms against the live
`GnssMeasurementsEvent` API.

## Headline results (synthetic ground-truth scenarios)

Generated by `apps/simulate_attack.py` (13 satellites visible, receiver moving east at 6 m/s,
a subset spoofed mid-track with a single-antenna meaconing model). Errors are horizontal RMS vs
ground truth, *during* the attack:

| Scenario | Satellites spoofed | Detection | False alarms | ID precision | Naive error | **Mitigated error** | Improvement |
|---|---|---|---|---|---|---|---|
| Subset, instant | 4 / 13 | 100 % | 0 % | 100 % | 312 m | **0.8 m** | **383×** |
| Subset, slow walk-off | 4 / 13 | 100 % | 0 % | 100 % | 215 m | **0.9 m** | **253×** |
| **Majority** spoof | 8 / 13 | 100 % | 0 % | n/a¹ | 305 m | **1.2 m** | **266×** |

¹ Under a *majority* spoof the spoofed set *is* the consensus, so it cannot be named geometrically;
the prior-gate + dead-reckoning fallback protects the position instead. This limitation is
fundamental to consensus methods and is handled explicitly (see *Design*).

Reproduce:

```bash
python apps/simulate_attack.py                                   # slow walk-off subset
python apps/simulate_attack.py --no-ramp --offset 600,200,0      # instant subset
python apps/simulate_attack.py --no-ramp --spoof G03,G06,G07,G10,G13,E03,E04,E08 --offset 500,200,0
```

## Validation on real Android recordings

Run on real multi-constellation GnssLogger captures (GPS+GLONASS+Galileo+BeiDou+QZSS),
scored against the phone's own NMEA solution as reference:

| Recording | Epochs | Standalone PVT vs NMEA (horizontal) | Spoof verdicts |
|---|---|---|---|
| Suburban, ~5 min | 280 | **median 5.0 m, RMS 7.4 m, P95 14 m** | 0 SPOOFED (169 CLEAN / 111 SUSPECT) |
| Urban, ~44 min | 2628 | median 9.3 m, P95 15 m | 0.1 % SPOOFED |
| Urban, ~43 min | 2590 | median 6.1 m, P95 10 m | 0.8 % SPOOFED |

The standalone position (computed purely from raw pseudoranges, never the OS location)
tracks the device's own fix to a few metres — and on clean data the detector raises **no false
spoofing alarms** on the good recording, flagging only routine multipath as SUSPECT while keeping
the position. The urban recordings have a heavy-multipath tail (rare km-level excursions) where the
engine dead-reckons to protect the track; the few SPOOFED flags there are multipath reflections that
momentarily mimic a coherent subset, not real attacks (no recording in this set was spoofed).

```bash
python apps/fetch_brdc.py 2026-03-21                                          # ephemeris for the day

python apps/run_realtime.py --obs your_recording.26o --nav data/brdc/BRDC_2026_080.rnx
```

## Quick start

Requires **Python 3.10+**. No third-party packages.

```bash
# 1. Run the end-to-end spoofing demo with ground truth (no data needed)
python apps/simulate_attack.py

# 2. Run the engine on a real recording (Android GnssLogger CSV or RINEX)
python apps/fetch_brdc.py 2026-06-18                             # get matching ephemeris
python apps/run_realtime.py --gnsslog data/raw/your_log.csv --nav data/brdc/BRDC_2026_169.rnx
python apps/run_realtime.py --obs data/raw/your_rec.26o --nav data/brdc/BRDC_2026_169.rnx

# 3. Run the test suite
python tests/run_tests.py            # or: pytest tests/
```

Outputs land in `outputs/`: a per-epoch CSV (trusted PVT + spoof status + diagnostics), a KML
showing the **trusted (green)** vs **naive (red)** tracks with spoof markers, and a spoof-event log.

## How it works

```
 raw measurements                 standalone PVT                 anti-spoofing engine
 ────────────────                 ──────────────                 ────────────────────
 Android GnssLogger CSV ─┐                                    ┌─ physical: AGC / C-N₀  (RPM)
                         ├─► unified EpochObs ─► sat states ──┼─ geometric: RANSAC consensus
 RINEX 3/4 OBS ──────────┘     (pseudorange,     (ECEF, clk)  ├─ geometric: RAIM FD/FDE
                                C-N₀, Doppler,                 └─ trust anchor: prior gate + DR
                                AGC)                                      │
                                                                         ▼
                                                        status + named spoofed PRNs +
                                                        trusted (mitigated) PVT  →  CSV/KML
```

### 1. Standalone positioning from raw measurements
- **`src/io/gnsslogger.py`** reconstructs pseudoranges from the Android raw hardware-clock fields
  (`TimeNanos`, `FullBiasNanos`, `ReceivedSvTimeNanos`, …) exactly per Google's
  `gps-measurement-tools` and the EUSPA white paper, with per-constellation time-base handling
  (GPS/Galileo week, BeiDou −14 s, GLONASS time-of-day) and `State`-bitmask validity filtering.
- **`src/core/solver.py`** is the reusable Ex0 solver, refactored: it computes satellite ECEF
  positions/velocities and clock corrections from broadcast ephemeris (Keplerian + GLONASS RK4,
  relativistic + TGD + Sagnac corrections) and runs **weighted least squares** for receiver
  position, clock, and one **inter-system bias** per constellation, plus velocity from Doppler.
- Because the position is computed *only* from RF observables, OS-level / network-layer spoofing
  (mock locations, rogue Wi-Fi) is bypassed entirely.

### 2. Detecting spoofing of a subset
- **RANSAC (`src/antispoof/ransac.py`)** — estimates the inter-system biases once, *whitens* every
  pseudorange into a single-clock frame (keeping the minimal subset at **4** across all
  constellations), then fits PVT to many minimal subsets and keeps the largest self-consistent
  (consensus) set. The excluded set is the spoofed subset. Inlier gating is elevation-weighted
  (k·σ / sin el); quartets are enumerated when cheap, randomly sampled with
  `N = ⌈log(1−p)/log(1−wˢ)⌉` otherwise.
- **RAIM (`src/antispoof/raim.py`)** — classical residual integrity: the weighted
  sum-of-squared-residuals `SSE = wᵀWw` is tested against a **chi-square** threshold
  `χ²₍₁₋Pfa₎(n−m)` (a dependency-free chi-square implementation lives in `stats.py`), with
  Fault-Detection-and-Exclusion that removes the largest normalised residual and re-tests.
- **Physical RPM (`src/antispoof/physical.py`)** — learns an open-sky AGC/C-N₀ baseline, then flags
  an **AGC drop** (added RF power) and **abnormally high / uniform C-N₀** (single-antenna power),
  classified by the Receiver-Power-Monitoring truth table (jamming vs spoofing).

### 3. Overcoming it
- **`src/antispoof/detector.py`** fuses the three: a geometrically isolated, RAIM-consistent inlier
  set with corroborating evidence ⇒ **SPOOFED**, spoofed PRNs named, trusted PVT from the inliers.
  A pure physical alarm without a separable subset still raises **SPOOFED** but holds position by
  dead reckoning (it does not trust the possibly-biased all-satellite fix).
- **`src/realtime/engine.py`** streams epochs, carries the last trusted position **propagated by
  velocity** (so the prior never goes stale), and uses it as a **gate**: a consensus that jumps
  beyond the gate is rejected as a likely majority spoof and the engine dead-reckons. This is the
  trust anchor that the literature shows RANSAC needs to survive a majority attack.

## The spoofing simulator (ground truth)

Real spoofed recordings are scarce, so `src/simulate/` builds controllable ones with known truth:

- **`scenario.py`** — a synthetic Walker GPS+Galileo constellation and a forward measurement model
  that *reuses the solver's own satellite routines*, so the solver inverts it to <1 mm (noise-free);
- **`spoofer.py`** — replaces a chosen subset of satellites with pseudoranges self-consistent with a
  *false* position (instant or slow walk-off), and adds the AGC-drop / high-C-N₀ RF signature;
- **`gnsslogger_writer.py`** — writes the result as a real **GnssLogger CSV**, so the simulator
  exercises the exact same parser a real phone log flows through.

## Android app (`android/`)

A Kotlin module that mirrors the Python engine against the live Android API:
`GnssAntiSpoofEngine` consumes `GnssMeasurementsEvent` (raw pseudoranges + `Cn0DbHz` +
`AutomaticGainControlLevelDb`), runs the same pseudorange reconstruction, WLS, RANSAC, RAIM and RPM
logic, and exposes a standalone trusted location plus a spoofing status — alongside the OS
`FusedLocationProvider` location for comparison. It is provided as a faithful, documented port
(it is not compiled in this repo). See [`android/README.md`](android/README.md).
